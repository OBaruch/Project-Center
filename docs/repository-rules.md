# Repository Rules

## Core rules

1. Markdown is the source of truth.
2. One project, one file.
3. Public repo means no secrets.
4. Keep summaries factual and readable.
5. Update the index when you add important content.
6. Prefer durable language over temporary notes.
7. Keep structure stable so the repo remains reusable.

## Recommended formatting

- Use H1 once per file.
- Use clear H2/H3 structure.
- Use bullet lists for scanability.
- Keep line lengths readable.
- Use relative links whenever possible.

## Public-private separation

If needed, maintain two systems:

- a **public Project Center** for presentation,
- a **private internal knowledge repo** for sensitive information.

## Maintenance cadence

Recommended review rhythm:

- monthly: update active projects,
- quarterly: refresh summary and CV,
- yearly: archive outdated items and refine public positioning.
