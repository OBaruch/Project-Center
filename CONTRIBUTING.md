# Contributing

This repository is designed to be cloned and adapted as a personal project portfolio system.

## How to use this template

1. Fork or clone the repository.
2. Replace placeholder content with your own information.
3. Keep the Markdown structure consistent.
4. Avoid uploading confidential information.
5. Prefer one file per project.

## Documentation rules

- Use clear project names.
- Use lowercase kebab-case for project filenames.
- Keep summaries concise and specific.
- Separate facts from reflections.
- Prefer structured sections over unformatted notes.

## Project naming convention

Use:

```text
projects/project-name.md
```

Examples:

- `projects/ai-finance-dashboard.md`
- `projects/vps-security-audit.md`
- `projects/personal-brand-site.md`

## Commit recommendations

Recommended commit styles:

- `docs: add new project file`
- `docs: update master cv`
- `docs: refine profile summary`
- `docs: update project index`
- `docs: add timeline entries`

## Public safety note

Never publish:

- secrets,
- API keys,
- confidential client data,
- private chats without review,
- internal company material,
- legal or tax documents that should remain private.
